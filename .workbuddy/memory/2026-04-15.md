### 核心架构（用户指正的关键点）

**三层 expect/actual 结构**：
```
commonMain:  expect object + actual external fun 声明
jvmMain:     actual object : IKniRegister + onRegister { ::func.register(...) }
nativeMain:  actual object + loader() + actual external fun 实现
```

**关键机制**：
1. `::functionName.register(staticCFunction { ... })` - 自动将函数映射到 Java 同名类同名方法
2. `asJni` / `asKni` - Kotlin 与 Java 对象互转
3. `KniAny` - 包装 Java 对象，支持序列化/反序列化
4. `"java.lang.Class".toClass()` - 类名字符串转 Java Class
5. `JniCallbackFactory` - Java 回调转 Kotlin
6. `KniCallbackFactory` - Kotlin Lambda 转 Java 回调
7. `kniResultJava { }` - 自动转换返回值为 Java 对象

### 关键文件
- `KniBridge.kt` - 核心桥接类（56KB）
- `KniRegister.kt` - 方法注册（支持 Function0-12 和 KFunction0-12）
- `KniVM.kt` - JNI_onLoad/onUnload 生命周期
- `IKniRegister.kt` - 注册接口
- `KniAny.kt` - Java 对象包装与序列化
- `KniCallbackFactory.kt` / `JniCallbackFactory.kt` - 回调转换
- `VariableTypeFactory.kt` - 类型快捷访问

### Android IPC
- `DreamSmartIPC.kt` - 入口
- `DreamJsonBinder` / `DreamSharedMemoryBinder` / `DreamJsonChunkBinder` - 三种传输模式

### 普通实现 vs 内联实现（深入理解）

**普通实现**：
```kotlin
::create.register(staticCFunction { _, _ -> create() })
actual fun create(): Long = MemoryTool.createSave().hashCode().toLong()
```
逻辑写在 actual 函数中，注册回调调用它。

**内联实现**：
```kotlin
::create.register(staticCFunction { _, _ -> MemoryTool.createSave().hashCode().toLong() })
actual fun create(): Long = notImplemented()
```
::functionName 只用于获取类名/函数名/参数类型，逻辑直接在回调中。

### kniResultJava 使用场景

- this 是 KniBridge，只有在 KniBridge 范围内才能调用 asJni/asEnum 等方法
- 内部使用 tryLocalFrame，自动 fPopLocalFrame 清理
- 需要转换返回值为 Java 对象时使用
- 返回简单类型（如 jstring）不需要用它

### 修正：简单类型定义

**简单类型**：Kotlin/Native 和 Kotlin/JVM 类型一致，不需要 JNI 转换
- Boolean, Char, Int, Long, Float, Double, Byte, Short

**非简单类型**：需要 JNI 函数操作
- jstring（需要 GetStringUTFChars）
- jobject, jclass, jarray 等
- 所有 Java 对象引用

所以 `jstring` 也不是简单类型，需要 JNI 环境。

### IPC 规则补充（第三次指正）

**RootService 只是其中一种暴露方式**：
- 普通 Service：不需要 root 权限
- RootService：需要 root 权限（如操作其他进程内存）

**IPC 接口限制**：
- 必须继承 IDreamIPC
- 必须是真正的接口（不能用普通类）

**参数/返回值序列化规则**：
1. 可序列化类型 → JSON 序列化（Int、String、@Serializable data class）
2. 接口类型 → 代理传输（ICallback、IUser）
3. 无法序列化且无接口 → ❌ 抛出错误

**asClient vs asClientAutoProxy**：
- asClient：需要手动处理返回值代理
- asClientAutoProxy：自动代理所有参数和返回值（推荐）

### README 核心概念重写（第四次指正）

**三层结构精确角色**：
- commonMain：声明需要桥接的类、函数、属性
- jvmMain：注册，在 init 加载 Native 库，actual external fun 空实现
- nativeMain：实现 IKniRegister，在 onRegister 中绑定 JNI 回调

**:: 双冒号多种用法**：
- `::函数` - 基本引用
- `this@OuterClass::函数` - 指定 this 域
- `类名::函数` - 类级别引用
- `val fun: KFunction1<Int, Int> = ::函数` - 指定类型的 KFunction
- `val fun: (Int, Int) -> Int = ::函数` - 需要 `.asKFunction().register()`

**泛型函数处理**：
- 直接 `::泛型函数` 会报错 "Cannot infer type for type parameter"
- 需要显式指定类型或通过属性类型推断

**KniVM.onLoad 默认注册**：
- KniCallbackProxy：Kotlin → Native 回调代理
- KniLogger：日志处理

### kni vs kniResultJava（第五次指正）

| 函数 | 用途 | 返回值处理 |
|------|------|----------|
| `kni {}` | 只使用 asString 等转换，无需 asJni | 返回值不会被 PopLocalFrame 清理 |
| `kniResultJava {}` | 需要将返回值转换为 Java 对象 | 自动 asJni，返回值不会被 PopLocalFrame 清理 |

两者内部都使用 tryLocalFrame 管理局部引用。

### Kni 反射能力重写（第六次指正）

之前写的 `bridge.method {}` 太概念化了。

**简洁反射 API**：
```kotlin
"类名".toClass().method { name = "方法名" }.string()
// 等价于 Java: clazz.getMethod("方法名").invoke(obj)

"类名".toClass().field { name = "字段名" }.string()
// 等价于 Java: clazz.getField("字段名").get(obj)

// 支持 thisRef 设置实例引用
// 支持 modifiers 设置静态/最终等
// 支持 param 设置参数类型
// 返回值有 .string()/.int()/.long()/.boolean() 等简写

### 反射注意事项（第七次指正）

反射通过 JNI GetMethodID/GetStaticMethodID 实现，必须写全类型：

1. 类名必须是全限定名（"java.lang.System" 而非 "System"）
2. 方法名必须完全匹配（大小写敏感）
3. 参数类型必须是 JNI 签名格式
4. 使用描述符更可靠：`"Ljava/lang/String;->substring(II)Ljava/lang/String;"`
5. 找不到方法会直接报错

### KniBridge Kotlin 反射扩展（第八次指正）

`KFunction<*>.asKniMethod()` - 将 Kotlin 函数自动映射为 JNI 方法
`KProperty<*>.asKniField()` - 将 Kotlin 属性自动映射为 JNI 字段

自动提取 declaringClass、name、param、returnType，无需手动配置。

### 参数类型支持（第九次指正）

param() 支持多种类型，自动转换：
- 字符串类名：`"java.lang.String"` → toClass()
- KClass：`Int::class` → 自动识别基本类型
- KType：`property.returnType`
- KniClass：`StringClass`

KClass 自动转换规则：
- `Int::class` → `int`（JNI 基本类型）
- `String::class` → `java.lang.String`

移除：KniField 没有 `modifiers` 属性
